Budget Guy — Desktop Responsive Mockups

These mockups show how the app could look on a ~1200px desktop viewport with a persistent sidebar nav.
The big idea: Budget and Plan pages show multiple months side-by-side instead of just widening one month.
Transactions switch to a proper data table. Forms stay centered. Mobile is untouched.

Page 2 — Budget (Multi-Month View)

The big idea: On desktop, show 3 months side-by-side (Feb → Mar → Apr). Categories are pinned on the left. Each month gets Budgeted / Spent / Available columns. The current month is highlighted. You can edit budgeted amounts for any visible month inline. Future months show projected/default amounts in italic. A toggle lets you switch between 1, 2, or 3 months. On mobile, nothing changes — single month view with the left/right arrows.

How the responsive behavior breaks down across device sizes.

Breakpoint Strategy

Mobile (< 768px) — No Changes
Bottom tab bar. Full-width content. BottomSheets for pickers.
Card-based transactions. Single month budget view. FAB above nav.
Tablet (768px – 1023px)
Bottom tab bar still. Content gets wider padding.
Budget shows 2 months side-by-side.
Transactions stay as cards but with more info visible.
BottomSheets become centered modals. Account cards in 2-col grid.
Desktop (1024px+)
Sidebar nav (200px) replaces bottom tabs.
Budget shows 3 months side-by-side.
Plan shows 3 projection months + defaults column.
Transactions switch to table view. Category detail goes side-by-side.
Forms centered at max-w-lg. Toolbar buttons replace 3-dot menus.
KEY PRINCIPLE
Mobile stays untouched. All desktop changes use Tailwind md: and lg: prefixes. The multi-month view is the reason to use the desktop version — not just "wider mobile."
BACKEND IMPACT
Budget controller currently returns data for 1 month. Multi-month view needs a new endpoint or parameter (?months=3) that returns data for adjacent months. Same query logic, just run 3x. Plan projections schema already supports multiple indices — just need UI.
